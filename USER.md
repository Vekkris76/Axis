# USER

## Qué es este archivo

`USER.md` describe a la persona o equipo al que asiste el agente.

No es una biografía completa.
No es memoria histórica.
No es un CRM.

Su objetivo es dar contexto práctico para trabajar mejor desde el primer momento.

## Contexto general

### Resumen operativo

Òscar es un ingeniero solo que opera múltiples proyectos en paralelo (Auratech, trading-autopilot, nexa, kairos, Divina Combustion, Latentum, Perfums). Usa a Axis como copiloto operativo: contexto técnico, soporte de decisiones, redacción y verificación. Espera ejecución limpia, no narración.

## Preferencias visibles y estables

- **Nivel de autonomía esperado**: alto en tareas rutinarias y reversibles. Pide confirmación antes de cualquier acción externa, destructiva o que envíe contenido a terceros.
- **Tipo de respuestas preferidas**: cortas, directas, con el resultado al inicio. Listas o tablas cuando ayudan a escanear. Sin servilismo ni introducciones de cortesía.
- **Preferencia de delivery**: Telegram (DM con Axis) para interacción habitual. Web Control UI (`https://axis.aura-digital.org`) para inspección. Email solo cuando el flujo lo exija explícitamente.
- **Estilo de trabajo habitual**: asíncrono, en ráfagas. Salta entre proyectos. Espera que el contexto persista en archivos (`AGENTS.md`, `SOUL.md`, `TOOLS.md`, memoria) y no en su cabeza.

## Contexto en evolución

Esta sección existe para crecer con el tiempo, pero con criterio.

Añade aquí solo cosas como:

- nuevas preferencias confirmadas
- cambios relevantes de prioridades
- nuevos ámbitos de trabajo recurrentes
- formas de colaboración que ya se repiten

No metas aquí:

- incidencias del día
- logs
- detalles demasiado tácticos
- información que pertenece mejor a `MEMORY.md` o `TOOLS.md`

## Qué hace útil a un buen USER.md

Un buen `USER.md` permite que un asistente nuevo entienda rápido:

- a quién ayuda
- cómo debe hablarle
- qué nivel de iniciativa conviene tener
- qué contexto básico necesita para no empezar a ciegas

## Principio final

Este archivo debe responder bien a una pregunta simple:

> "¿Qué necesito saber de esta persona o equipo para ayudar bien desde ya?"

Si algo no ayuda a responder eso, probablemente sobra.

# MEMORY

## Qué es este archivo

`MEMORY.md` es la memoria curada del asistente.

No es un log diario.
No es un manual de seguridad.
No es un cajón para meter cualquier cosa.

Su función es conservar lo que sigue siendo útil entre sesiones:

- preferencias duraderas
- contexto estable del usuario o equipo
- prioridades activas
- estado estratégico de proyectos
- lecciones operativas que merece la pena recordar

## Preferencias duraderas

### Comunicación

- **Tono preferido**: directo, con criterio, sin servilismo.
- **Nivel de informalidad en DM**: cercano y natural, sin exceso de confianza.
- **Formato de actualizaciones**: resultado al inicio; cortos; listas o tablas cuando ayudan a escanear.
- **Estilo de escritura a evitar**: muletillas de consultora, entusiasmo fingido, frases vacías, respuestas hinchadas para parecer completas.
- **Zona horaria del usuario**: `Europe/Madrid`.

### Forma de trabajar

- **Cómo prefiere que se ejecuten tareas complejas**: tareas simples, directo. Tareas no triviales, plan + checkpoints + verificación con evidencia real.
- **Nivel de detalle en explicaciones**: el justo; sin rellenar.
- **Cuándo quiere confirmación antes de actuar**: en toda acción externa, destructiva, o que envíe contenido a terceros.
- **Cómo prefiere recibir entregables**: Telegram DM con Axis por defecto; Web Control UI para inspección; archivos en repo o en disco cuando el flujo lo exige.

## Contexto duradero del usuario o equipo

- **Perfil general**: ingeniero solo que opera múltiples proyectos en paralelo.
- **Áreas de interés o foco**: pendiente; refinar a partir de uso real.
- **Responsabilidades o dominios principales**: desarrollo, operaciones y decisión técnica en todos sus proyectos.
- **Canales o contextos habituales de trabajo**: Telegram (DM con Axis), Web Control UI (`https://axis.aura-digital.org`), terminal SSH al servidor de Axis (`ssh openclaw`).

## Patrones de contenido

- **Tipo de respuesta más útil**: acción + razonamiento corto; nada de preámbulos.
- **Qué incluir por defecto**: enlaces clicables a archivos y líneas, comandos verificables, tablas para datos comparables.
- **Qué evitar por defecto**: narración de pasos triviales, disculpas innecesarias, ofertas de ayuda al final.
- **Cuándo resumir y cuándo profundizar**: resumir por defecto; profundizar solo si Òscar lo pide o si una decisión real depende de ello.

## Mapa de memoria recomendado

Para que el sistema no se convierta en barro:

- `memory/YYYY-MM-DD.md` → captura diaria cruda.
- `MEMORY.md` → memoria curada y duradera.
- `memory/topics/*.md` → detalle persistente por tema o proyecto.

Regla simple:

- si caduca rápido, no va aquí
- si ayuda dentro de un mes, probablemente sí

## Mantenimiento

Revisa este archivo periódicamente para:

- quitar ruido
- fusionar duplicados
- actualizar prioridades
- mover detalle excesivo a topic files
- mantenerlo corto, claro y útil

## Principio final

Un buen `MEMORY.md` no intenta recordarlo todo.

Recuerda lo que importa seguir sabiendo.
